02 · Dasar Teori

Definisi & Sifat Dasar Pohon

  • 1Definisi (PDF Halaman 3): Pohon (*tree*) adalah graf tak-berarah terhubung yang tidak mengandung sirkuit atau siklus (*acyclic*).
  • 2Sebuah pohon dengan $n$ simpul (*vertex*) selalu memiliki tepat $n - 1$ sisi (*edge*) untuk menghubungkan seluruh simpulnya.
  • 3Setiap pasang simpul di dalam pohon terhubung dengan **lintasan tunggal** (*unique path*).
  • 4Penambahan satu sisi baru pada pohon akan membuat tepat satu sirkuit.
02 · Dasar Teori

Terminologi Pohon Berakar (Rooted Tree)

Sesuai dengan standar materi kuliah Anda di PDF Halaman 28-34:

Induk & Anak

Orangtua & Anak

Simpul di atas adalah orangtua (*parent*), simpul di bawahnya yang terhubung langsung adalah anak (*child*).

Ujung Cabang

Daun (Leaf)

Simpul yang tidak memiliki anak (derajat keluar = 0). Contoh pada graf adalah ujung terluar.

Kedalaman

Aras & Tinggi

Aras (*level*) dimulai dari 0 (akar). Tinggi (*height*) adalah aras maksimum dari suatu pohon berakar.

02 · Dasar Teori

Pohon Biner & Pohon Terurut

  • 1Pohon Biner (PDF Halaman 40): Adalah pohon n-ary dengan $n = 2$. Setiap simpul cabangnya memiliki paling banyak 2 anak.
  • 2Urutan Anak Penting: Dibedakan secara eksplisit antara **anak kiri** (*left child*) dan **anak kanan** (*right child*).
  • 3Pohon Terurut (PDF Halaman 36): Urutan anak-anak diatur secara spesifik. Anak kiri dan anak kanan tidak boleh tertukar.
  • 4BST menggabungkan kedua sifat ini untuk melakukan pemisahan data berdasarkan nilai numerik.
02 · Dasar Teori

Aturan Pengurutan Binary Search Tree

  • 1BST adalah pohon biner terurut dengan aturan perbandingan matematis.
  • 2Nilai subpohon kiri selalu lebih kecil ($<$) dari nilai simpul induk.
  • 3Nilai subpohon kanan selalu lebih besar ($>$) dari nilai simpul induk.
  • 4Aturan ini berlaku secara **rekursif** pada setiap subpohon di dalam struktur data tersebut.
02 · Dasar Teori

Operasi Penyisipan (Insert) pada BST

  • 1Operasi insert berfungsi memasukkan simpul data baru ke dalam struktur pohon.
  • 2Pencarian dimulai dari simpul **Akar** (*root*).
  • 3Bandingkan nilai baru: Jika **lebih kecil**, telusuri cabang **kiri**. Jika **lebih besar**, telusuri cabang **kanan**.
  • 4Ulangi terus langkah tersebut secara rekursif hingga menemukan posisi kosong (`null`), lalu sisipkan simpul baru.
Prinsip Insert

Pencarian Posisi Kosong

Data baru tidak pernah menimpa data yang sudah ada, melainkan selalu menjadi **daun baru** (*new leaf*) di ujung cabang pohon.

02 · Dasar Teori

Operasi Pencarian (Search) pada BST

Prinsip Pencarian

Pencarian Biner / Eliminasi

Pencarian pada BST mirip seperti algoritma *binary search*. Setiap langkah perbandingan akan mengeliminasi separuh subpohon yang tidak mungkin memuat nilai target.

  • 1Bandingkan nilai target dengan simpul saat ini (mulai dari akar).
  • 2Jika nilai target cocok, data **ditemukan** dan simulasinya selesai.
  • 3Jika target lebih kecil, pencarian bergerak secara eksklusif ke cabang **anak kiri**.
  • 4Jika target lebih besar, pencarian bergerak secara eksklusif ke cabang **anak kanan**.
02 · Dasar Teori

Penelusuran Pohon: Inorder Traversal

  • 1*Traversal* adalah proses mengunjungi setiap simpul di dalam pohon tepat satu kali.
  • 2**Inorder Traversal** menggunakan prinsip rekursif **LDR**: Kunjungi Kiri (*Left*), ambil simpul Induk (*Data*), lalu kunjungi Kanan (*Right*).
  • 3Pada BST, penelusuran ini secara matematis akan **selalu menghasilkan data yang terurut menaik** (*ascending*).
  • 4Sifat ini dimanfaatkan untuk melakukan penyortiran (*sorting*) data secara efisien.
Pola Kunjungan Simpul
Kiri → Induk → Kanan

Hasil output traversal selalu terurut dari nilai terkecil hingga terbesar.

03 · Implementasi

Metode Penyisipan (Insert) Rekursif

def insert(self, value): self.root = self._insert(self.root, value) def _insert(self, node, value): if node is None: return Node(value) # Buat simpul daun baru if value < node.value: node.left = self._insert(node.left, value) elif value > node.value: node.right = self._insert(node.right, value) return node
Alur Rekursi

Pencarian Posisi Kiri / Kanan

Fungsi pembantu `_insert` menerima parameter simpul saat ini, mengecek nilai, dan memanggil dirinya sendiri ke cabang kiri atau kanan hingga menemukan posisi kosong.

Metode Pencarian & Penelusuran (Inorder)

def search(self, value): current = self.root comparisons = 0 while current is not None: comparisons += 1 if value == current.value: return True, comparisons elif value < current.value: current = current.left else: current = current.right return False, comparisons
def inorder(self, node, result): if node is not None: self.inorder(node.left, result) result.append(node.value) self.inorder(node.right, result) return result # Fungsi search berjalan secara iteratif (perulangan) # Fungsi inorder berjalan secara rekursif (LDR)

04 · Hasil & Pembahasan

Analisis Kompleksitas & Keseimbangan

  • 1Kasus Rata-rata: Bila pohon dalam keadaan seimbang, kompleksitas pencarian data adalah **$O(\log n)$**.
  • 2Kasus Terburuk: Bila data dimasukkan sudah terurut (misal: 10, 20, 30), pohon akan condong (*skewed*) menyerupai *linked list* dengan kompleksitas **$O(n)$**.
  • 3Kinerja BST sangat bergantung pada aspek **Tinggi/Kedalaman Pohon**.
  • 4Untuk menjamin performa $O(\log n)$, aplikasi nyata memanfaatkan tipe pohon seimbang seperti **AVL Tree** atau **Red-Black Tree**.
