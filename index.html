01 · Latar Belakang

Pencarian data linear tidak lagi efisien

  • 1Volume data yang dikelola sistem terus bertambah besar setiap saat.
  • 2Pencarian linear memeriksa data satu per satu — kompleksitas O(n), lambat pada data besar.
  • 3Struktur pohon (tree) menawarkan cara menyimpan data yang mendukung pencarian jauh lebih cepat.
  • 4Binary Search Tree (BST) memanfaatkan sifat terurut untuk mempercepat insert, search, dan delete.
Perbandingan Kasar
O(log n)

vs pencarian linear O(n) — makin besar data, makin terasa selisihnya.

03 · Dasar Teori

Pohon (Tree): struktur hierarkis tanpa siklus

  • •Root — simpul teratas, tidak memiliki induk.
  • •Parent / Child — simpul yang terhubung langsung secara bertingkat.
  • •Leaf — simpul tanpa anak, berada di ujung cabang.
  • •Pohon dengan n simpul selalu memiliki tepat n − 1 sisi, dan tidak memuat siklus.
03 · Dasar Teori

Binary Search Tree: aturan pengurutan

  • •Setiap simpul memiliki paling banyak dua anak — kiri dan kanan.
  • •Nilai subpohon kiri selalu lebih kecil dari simpul induk.
  • •Nilai subpohon kanan selalu lebih besar dari simpul induk.
  • •Sifat ini berlaku rekursif pada setiap subpohon — karena itu BST disebut juga pohon pencarian.
03 · Dasar Teori

Tiga operasi inti pada BST

Operasi 01

Insert

Menyisipkan nilai baru dengan menelusuri pohon secara rekursif hingga menemukan posisi kosong yang sesuai aturan BST.

Operasi 02

Search

Membandingkan nilai target terhadap simpul saat ini, lalu bergerak ke kiri atau kanan hingga ditemukan.

Operasi 03

Inorder Traversal

Menelusuri kiri → simpul → kanan. Pada BST, hasilnya selalu berupa data terurut menaik.

04 · Implementasi

Inti program: kelas Node & fungsi Search

class Node: def __init__(self, value): self.value = value self.left = None self.right = None # insert: rekursif, kiri jika lebih kecil, # kanan jika lebih besar dari node saat ini
def search(self, value): current = self.root comparisons = 0 while current is not None: comparisons += 1 if value == current.value: return True, comparisons elif value < current.value: current = current.left else: current = current.right return False, comparisons

05 · Hasil & Pembahasan

Kenapa BST lebih cepat?

  • •Setiap perbandingan mengeliminasi separuh sisa data yang mungkin — kompleksitas rata-rata O(log n).
  • •Pencarian linear harus memeriksa data satu per satu — kompleksitas O(n) di semua kondisi.
  • •Keunggulan BST bergantung pada keseimbangan pohon. Data yang sudah terurut membuat BST menyerupai linked list (O(n)).
  • •Solusinya: varian pohon seimbang seperti AVL Tree atau Red-Black Tree pada aplikasi nyata.
06 · Kesimpulan

Tiga hal yang bisa dibawa pulang

01

Konsep Pohon

BST adalah pohon biner yang menerapkan aturan pengurutan kiri < simpul < kanan.

02

Terbukti Bekerja

Inorder traversal membuktikan sifat BST: hasil penelusuran selalu terurut menaik.

03

Lebih Efisien

BST membutuhkan jauh lebih sedikit perbandingan dibanding pencarian linear pada data uji.
